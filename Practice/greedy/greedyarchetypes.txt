Out of the five following greedy archetypes, what does this problem fall into?

🧩 The 5 Common Greedy Archetypes
Almost all greedy problems fall into these:

1️⃣ Activity Selection Type (Optimization by sorting, [intervals]) -> Interval sorting and exchange
Key: earliest finish time
Reason: leaves maximum room for future

2️⃣ Resource Allocation Type (Marginal gain, [weighted optimization])
Key: best marginal gain per cost
Example: fractional knapsack

3️⃣ Gap Cutting Type ([Gap] selection)
Key: pick largest/smallest adjacent gaps
Example: Kindergarten
[General Template for Gap-Type Problems]
1. Sort
2. Reduce cost to:
	- Sum of adjacent differences
	- Or sum of block maxima
3. Greedily remove or group largest contributions

4️⃣ Merge Smallest First
Key: combine smallest elements first
Example: Huffman coding

5️⃣ Stay Ahead Strategy (Prefix correctness invariant)
Key: keep partial solution always better than optimal’s prefix
Example: interval scheduling

[May use heap to store prefix]
