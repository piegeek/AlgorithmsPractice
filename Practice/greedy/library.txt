A.
In DP you compute cost:
abs(positions[last] - 0) + abs(0 - positions[i])

Meaning:
last → 0 → i

So path order should be:
positions[last] → 0 → positions[i] → ...

But you're appending:
0 → positions[last]

That is reversed.
